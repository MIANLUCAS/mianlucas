******************************* GIT / GITHUB  *****************************************

git init                        # Crear un repositorio en la carpeta actual
git clone [url-repository]      # Crear un repositorio basado en un repositorio remoto

## Manejo de ramas
git branch [branch-name]        # Crear una rama desde la rama actual
git branch -m [branch-name]     # Cambiarde nombre a la rama actual
git branch -d [branch-name]     # Eliminar una rama local
git checkout [branch-name]      # Moverse a una rama seleccionada
git checkout [commit-hash]      # Moverse a un commit seleccionado
git checkout -b [branch-name]   # Crear rama desde la rama actual y hacer checkout
git merge [branch-name]         # Unir la rama actual con la rama seleccionada

## Manejo de commits
git add [file-name]             # Agregar el archivo seleccionado al stage 
git add --all                   # Agregar todos los archivos modificados al stage
git reset [file-name]           # Descartar el archivo del stage 
git commit                      # Crear un commit, abre el editor para escribir
git commit -m "[description]"   # Crear un commit, con una descripcion breve

## Estado y consultas
git status                      # Lista de archivos modificados no versionados
git log                         # Muestra todos los commits de la rama actual
git log --oneline               # Muestra todos los commits (primera linea)

******************************* WP-CLI *****************************************

Los comandos siempre comienzan con wp seguido por un Comando y Subcomando, seguido por parámetros obligatorios y opcionales, algo como esto:

wp command subcommand requiredparam --optionalparam --optionalparam2=value

Instalar worpress

wp core download
wp core config --dbname=mydbname --dbuser=mydbuser --dbpass=mydbpass --dbhost=localhost --dbprefix=whebfubwef_ --extra-php <<PHP
define( 'WP_DEBUG', true );
define( 'WP_DEBUG_LOG', true );
PHP
wp db create
wp core install --url=http://siteurl.com --title=SiteTitle --admin_user=username --admin_password=mypassword --admin_email=my@email.com

Reinstalar Wordpress core
El siguiente comando bajaría WordPress sin temas, ni plugins por defecto.

wp core download --skip-content --force

Lista detallada de Plugins Actuales

wp plugin list

Instalar Plugin o vario plugins

wp plugin install advanced-custom-fields jetpack ninja-forms --activate


Instalar Tema

wp theme install sorbet --activate


Instalar versiones antiguas de los plugins para Wordpress

wp plugin install wordpress-seo --version=4.8 --activate

Desactivar Plugins
wp plugin deactivate wordpress-seo

wp plugin deactivate wordpress-seo jetpack (mas de un plugin)

wp plugin deactivate --all (asi desactivariamos todos los plugins)

Actualizar Plugins
wp plugin update wordpress-seo


Administrando Eventos WP-Cron
Podemos administrar eventos WP-Cron y/o un WordPress Cron job en WP-CLI. Por ejemplo, el siguiente comando nos dará una lista de eventos cron actual.

wp cron event list


Generar Datos de Prueba
Generaramos usuarios, términos y posts por ejemplo para hacer pruebas

wp user generate --count=5 --role=editor
wp user generate --count=10 --role=author
wp term generate --count=12
wp post generate --count=50


Borrar Revisiones WordPress
Las molestas revisiones pueden ser eliminadas desde el siguiente comando:
$ wp post delete $(wp post list --post_type='revision' --format=ids)

Pero tambien lo podemos hacer manualmente desde el archivo wp-config.php:
Limitar a 3 revisiones (El código siguiente debe ser insertado por encima de ‘ABSPATH’ de lo contrario no funcionará)
define('WP_POST_REVISIONS', 3);
Limitar todas las revisiones (El código siguiente debe ser insertado por encima de ‘ABSPATH’ de lo contrario no funcionará)
define('WP_POST_REVISIONS', false);
